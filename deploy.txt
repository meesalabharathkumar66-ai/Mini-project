# S.A.M SECURE VAULT - DEPLOYMENT GUIDE

## 1. PREPARING FOR MARKET (GIT UPLOAD)
1. Initialize Git repository:
   ```bash
   git init
   ```
2. Create a .gitignore file (Ensure node_modules and .env are included).
3. Add files and Commit:
   ```bash
   git add .
   git commit -m "Initialize S.A.M Production Build"
   ```
4. Push to GitHub/GitLab:
   ```bash
   git remote add origin YOUR_REPOSITORY_URL
   git push -u origin main
   ```

## 2. PRODUCTION DEPLOYMENT (MARKET LAUNCH)
We recommend using **Vercel** for the frontend and **Render** or **Railway** for the backend for a cost-effective/free market launch.

### A. Frontend (Vercel) - Recommended
1. Sign in to [Vercel](https://vercel.com).
2. Connect your GitHub repository.
3. Import the `web` directory.
4. Set Environment Variables:
   - REACT_APP_API_URL: Your deployed backend URL.
5. Deploy! Vercel provides a free global CDN and SSL.

### B. Backend (Render / Railway)
1. Import the `server` directory to Render or Railway.
2. Configure Node.js environment.
3. Set Environment Variables (Crucial):
   - MONGO_URI: Your production MongoDB Atlas string.
   - JWT_SECRET: A strong random string.
   - SESSION_SECRET: A strong random string.
   - GOOGLE_CLIENT_ID / SECRET: From Google Cloud Console.

### C. Database (MongoDB Atlas)
- Use MongoDB Atlas Free Tier for a reliable, managed cloud database.

## 3. FINAL SECURITY CHECKLIST

- [ ] Set storage limits for users in the database.
